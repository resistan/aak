# ABT-1.3.3-Processor: 명확한 지시사항 제공 v0.14

### 🔗 References
- KWCAG 2.2: 1.3.3 명확한 지시사항 제공
- WCAG 2.2: 1.3.3 Sensory Characteristics (A) (Reference)

---

이 알고리즘은 **KWCAG 1.3.3(명확한 지시사항 제공)** 지침을 준수하며, 정보나 조작 방법을 전달할 때 오직 시각적 형태, 크기, 위치 또는 소리 등 감각적 특징에만 의존하는 지시사항이 있는지 진단한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Text Analysis:** `p`, `span`, `div`, `li`, `label`, `h1`~`h6` 요소 내의 직접적인 텍스트 노드 콘텐츠 추출.
- **Pattern Repository:** 감각적 특징(위치, 색상, 형태, 소리)과 조작/안내 키워드가 결합된 정규표현식 패턴.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### 지능형 구문 패턴 매칭 (Pattern Matching)
단일 키워드(예: "위", "음성")만으로는 진단하지 않으며, 반드시 안내의 성격을 띠는 단어와 결합된 경우만 포착한다.

**탐지 패턴:**

1. **위치+지시:**
   - 패턴: `(왼쪽|오른쪽|위쪽|아래쪽|상단|하단|옆|측면|앞|뒤)` + `(버튼|링크|아이콘|메뉴|항목|탭|클릭|누르|선택|참조|확인|사용)`
   - 예시: "왼쪽 버튼을 클릭", "상단 메뉴 참조"

2. **색상+지시:**
   - 패턴: `(빨간|파란|노란|초록|검정|흰색|어두운|밝은)` + `(색|버튼|링크|아이콘|클릭|누르|항목)`
   - 예시: "빨간색 버튼", "초록색은 완료"

3. **형태+지시:**
   - 패턴: `(동그란|네모난|원형|사각형|둥근|모양|큰|작은)` + `(버튼|링크|아이콘|클릭|누르|항목)`
   - 예시: "동그란 아이콘", "큰 버튼"

4. **소리+지시:**
   - 패턴: `(소리|신호음|비프|벨|음성)` + `(로|를|가|이|들리면|나면|확인)`
   - 예시: "신호음이 들리면"

### 결과 판정
- **Rule 1.3.3 (Sensory Characteristics):** 패턴 매칭 시
  - 결과: **[검토 필요]**
  - 가이드: "지시사항에 감각적 표현이 포함된 구문이 탐지되었습니다. 시각이나 청각 등 특정 감각에만 의존하여 정보를 전달하고 있지 않은지 수동으로 검토하세요."

## 3. 최종 상태 정의 (Final Status)

1. **검토 필요:** 감각 표현과 지시어가 결합된 의심 구문이 발견된 경우.
2. **적절:** 감각적 특징을 언급하더라도 논리적인 이름(예: "왼쪽 [로그인] 버튼")을 함께 제공하여 정보 전달이 명확함.
3. **N/A:** 페이지 내에 지시/안내 성격의 텍스트 콘텐츠가 존재하지 않는 경우.
