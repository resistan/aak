# ABT-4.2.1-Processor: 웹 애플리케이션 접근성 준수 v0.11

### 🔗 References
- KWCAG 2.2: 4.2.1 웹 애플리케이션 접근성 준수
- WCAG 2.2: 4.1.2 Name, Role, Value (A) (Reference)

---

이 알고리즘은 **KWCAG 4.2.1(웹 애플리케이션 접근성 준수)** 지침을 준수하며, 웹 애플리케이션(Single Page Application, Rich Internet Application 등)이 표준 웹 접근성 지침을 자체적으로 준수하는지, 또는 접근 가능한 대체 수단을 제공하는지 진단한다. 이 지침은 광범위한 평가를 요구하므로, ABT 엔진은 잠재적 웹 애플리케이션 요소를 탐지하고 전문가의 종합적인 판단을 유도하는 가이드 역할을 수행한다.

> **배점 방식:** 전수 조사형 (Exhaustive Scan). 커스텀 ARIA 위젯을 수집하여 각각 판정하며, 비율 기반으로 점수를 산정한다. 위젯이 감지되지 않은 경우에도 전문가 확인 요청 리포트 1건을 생성한다.

## 1. 대상 식별 및 데이터 수집 (Data Collection)

### [대상 요소 추출]
- **커스텀 위젯 탐지**: `role` 속성이 부여된 주요 대화형 요소 (`button`, `link`, `checkbox`, `switch`, `tab`, `dialog`, `menuitem`, `combobox`).

### [접근성 메타데이터 수집]
- `aria-live`, `aria-atomic`, `aria-relevant` 등 동적 콘텐츠 업데이트 관련 ARIA 속성 사용 패턴.
- `role="alert"`, `role="status"`, `role="log"` 등 상태 메시지 제공 역할.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [검사 1] 커스텀 위젯 탐지 및 기본 속성 검사
- **Rule 4.2.1 (Custom Widget Review):** `role` 속성이 부여된 요소를 발견하면 목록화한다.
  - 결과: 발견 시 **[검토 필요]**
  - 가이드: "역할(role)이 부여된 커스텀 위젯입니다. 상태 변화가 스크린 리더에 잘 전달되고 키보드로 조작 가능한지 확인하세요."
- **Rule 4.2.1 (Missing Tabindex on Widget):** 대화형 역할을 가진 커스텀 요소가 `tabindex` 속성을 가지고 있는가?
  - 결과: 누락 시 **[수정 권고]**
  - 가이드: "역할(role)이 부여되었으나 초점을 받을 수 없습니다(tabindex 누락). 키보드 접근성이 심각하게 훼손되었을 가능성이 있습니다."

### [검사 2] 전문가 수동 검사 (종합)
- **Rule 4.2.1 (Manual ARIA Review):** 발견된 위젯들이 키보드로 조작 가능한지, 상태 변화(`aria-expanded` 등)가 적절히 전달되는지 확인하십시오.
  - 결과: 위젯 탐지 시 **[검토 필요]**
  - 가이드: "[수동 검사 안내] 커스텀 ARIA 위젯이 발견되었습니다. 이 위젯들이 1) 키보드만으로 조작 가능한지, 2) 상태 변화(aria-expanded, aria-checked 등)가 스크린 리더에 정확히 전달되는지 수동으로 검토하세요."
- **Rule 4.2.1 (No Custom Widgets):** 위젯이 하나도 없는 경우.
  - 결과: **[검토 필요]**
  - 가이드: "커스텀 ARIA 위젯이 감지되지 않았습니다. 페이지에 JavaScript로 동작하는 동적 컴포넌트가 있는지 확인하세요."

### [단계 C] 전문가 종합 판단 가이드라인
ABT 엔진은 웹 애플리케이션 특성을 가진 페이지를 탐지하고, 전문가에게 다음 체크리스트를 제공하여 종합적인 접근성 준수 여부를 판단하도록 유도한다.
- *체크리스트*:
  - **자체 접근성 확인**:
    - 키보드만으로 모든 기능 조작이 가능한가? (6.1.1, 6.1.2 등 재확인)
    - ARIA 랜드마크, 역할, 상태, 속성이 의미론적으로 올바르게 사용되었는가?
    - 동적 콘텐츠 업데이트 시 보조기술 사용자에게 적절히 알림이 제공되는가?
    - 페이지 상태 변경(SPA 라우팅) 시 `<title>` 등 정보가 업데이트 되는가?
  - **대체 콘텐츠 제공 확인**:
    - 웹 애플리케이션이 자체적으로 접근성을 갖추기 어렵다면, 동일한 정보를 제공하는 대체 콘텐츠(예: 일반 HTML 버전)가 제공되는가? (예: 모바일 앱과 웹 앱이 별도로 존재)

## 3. 최종 상태 정의 (Final Status)
1. **수정 권고 (Suggestion):**
   - 커스텀 대화형 위젯에 `tabindex`가 누락되어 키보드 접근이 불가능한 경우.
2. **검토 필요 (Needs Review):**
   - 커스텀 ARIA 위젯이 탐지되어 전문가의 동작 확인이 필요한 상태.
   - 위젯이 감지되지 않아 JavaScript 동적 컴포넌트 존재 여부를 전문가가 직접 확인해야 하는 경우.
3. **적절:**
   - 전문가가 커스텀 위젯의 키보드 조작 및 상태 전달이 올바르게 구현되었음을 확인한 경우.
