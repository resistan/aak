# ABT-2.4.3-Processor: 적절한 링크 텍스트 v0.11

### 🔗 References
- KWCAG 2.2: 2.4.3 적절한 링크 텍스트
- WCAG 2.2: 2.4.4 Link Purpose (In Context) (A) (Reference)

---

이 알고리즘은 **KWCAG 2.4.3(적절한 링크 텍스트)** 지침을 준수하며, 링크의 텍스트가 목적을 명확히 설명하고 있는지 진단한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Accessible Name:** `innerText`, `aria-label`, `title`, 이미지 `alt` 등을 종합한 링크의 이름.
- **Descriptions:** `aria-describedby`로 연결된 보조 설명 텍스트.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 링크 이름의 유효성 검사
- **이름 부재:** 링크의 목적을 알 수 있는 텍스트가 전혀 없는 경우.
  - 결과: **[오류]**.
- **모호한 표현:** '여기', '클릭', '더 보기' 등 모호한 단어 사용 시.
  - `aria-describedby`가 있으면: **[검토 필요]**.
  - 없으면: **[부적절]**.
- **URL 노출:** 링크 텍스트로 기계적인 URL(http...)이 노출되는 경우.
  - 결과: **[수정 권고]**.

## 3. 최종 상태 정의 (Final Status)
1. **오류:** 링크 이름이 아예 없는 경우.
2. **부적절:** 맥락 정보 없이 모호한 단어만 사용된 경우.
3. **검토 필요:** 모호한 단어이나 보조 설명(`aria-describedby`)이 제공되어 확인이 필요한 경우.
4. **수정 권고:** 서술적 문구 대신 URL이 노출된 경우.
5. **적절:** 목적이 명확한 링크 텍스트가 제공됨.
