# ABT-3.3.3-Processor: 접근 가능한 인증 v0.11

### 🔗 References
- KWCAG 2.2: 3.3.3 접근 가능한 인증
- WCAG 2.2: 3.3.7 Accessible Authentication (Minimum) (AA) (Reference)

---

이 알고리즘은 **KWCAG 3.3.3(접근 가능한 인증)** 지침을 준수하며, 사용자 인증 과정(로그인, 본인 확인 등)이 비밀번호 기억, 복잡한 캡차(CAPTCHA) 풀이 등 인지 기능에 과도하게 의존하는 경우를 탐지한다.

> **배점 방식:** 전수 조사형 (Exhaustive Scan). 패스워드 필드 또는 auth 신호 기반으로 요소를 수집하여 각각 판정하며, 비율 기반으로 점수를 산정한다. 아무것도 감지되지 않은 경우에도 전문가 확인 요청 리포트를 1건 생성한다.

## 1. 데이터 수집 단계 (Data Collection)

### 1차: 패스워드 필드
- `input[type="password"]` (비가시 요소 제외)

### 2차: Auth 신호 탐지 (패스워드 필드 없을 때)
- **폼 action:** `login`, `auth`, `signin`, `oauth`, `sso` 키워드 포함
- **링크·버튼 href:** `/login`, `/auth`, `/signin`, `/oauth`, `/sso`, `kakao`, `naver`, `google`, `github` 키워드 포함
- **링크·버튼 텍스트:** `로그인`, `인증`, `sign in`, `log in`, `signin`, `login` 키워드 포함

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [경우 1] 패스워드 필드 존재

| 조건 | 상태 | Rule |
|---|---|---|
| `autocomplete="off"` 또는 `onpaste` 방지 | 오류 | Rule 1.1 / Rule 1.2 |
| 기술적 장벽 없음 | 검토 필요 | Rule 1.3 |

- **Rule 1.1:** `autocomplete="off"` → 패스워드 매니저 차단. 허용하도록 수정 필요.
- **Rule 1.2:** `onpaste="return false"` → 붙여넣기 차단. 허용하도록 수정 필요.
- **Rule 1.3:** 기술적 장벽 없으나 캡차 등 수동 확인 필요.

### [경우 2] 패스워드 없음 + Auth 신호 감지

- **Rule 2.1:** 소셜 로그인, OAuth 등 대체 인증 방식 페이지. 인지 능력에만 의존하지 않는 수단이 제공되는지 확인 유도.
- 결과: **검토 필요** (페이지 단위 리포트 1건)

### [경우 3] 아무것도 감지 안됨

- **Rule 2.2:** 자동 탐지 실패. 전문가에게 직접 확인 요청.
- 결과: **검토 필요** (페이지 단위 리포트 1건, 메시지: "이 페이지에 인증 과정이 있는지 확인하세요.")

## 3. 최종 상태 정의 (Final Status)

1. **오류:** 패스워드 필드에 기술적 장벽(autocomplete off, 붙여넣기 차단)이 있는 경우.
2. **검토 필요:** 인증 요소가 감지되었으나 대체 수단 유효성을 자동 판단할 수 없는 경우. 또는 감지 실패로 전문가 직접 확인이 필요한 경우.
3. **적절:** 전문가가 인지에 의존하지 않는 대체 인증 수단이 충분히 제공됨을 확인한 경우.
