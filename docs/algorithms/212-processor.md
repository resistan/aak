# ABT-2.1.2-Processor: 초점 이동과 표시 v0.14

### 🔗 References
- KWCAG 2.2: 2.1.2 초점 이동과 표시
- WCAG 2.2: 2.4.3 Focus Order (A) (Reference)

---

이 알고리즘은 **KWCAG 2.1.2(초점 이동과 표시)** 지침을 준수하며, 초점 이동의 논리성(`tabindex`)과 시각적 명확성(`outline` 가시성)을 진단한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Focusable Elements:** `a[href]`, `button`, `input`, `select`, `textarea`, `[tabindex]:not([tabindex="-1"])`, `details`, `summary`.
- **Styles:** 인라인 `style` 속성 내의 `outline` 정의.
- **Geometry:** `getBoundingClientRect()`를 통한 요소의 시각적 위치.
- **Count:** 숨김 요소(`isHidden`)를 제외한 포커스 가능 요소 총 개수 산출 → 수동 검사 안내에 활용.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [검사 1] 인라인 아웃라인 제거 탐지
- **Rule 2.1.2 (Focus Visibility):** 요소에 `style="outline: none"` 또는 `style="outline: 0"`이 직접 적용된 경우
  - 결과: **[오류]**
  - 가이드: "접근성에 치명적이므로 제거해야 합니다."

### [검사 2] 양수 Tabindex 탐지
- **Rule 2.1.2 (Focus Order):** `tabindex > 0`인 요소가 문서 중간/하단에 위치한 경우
  - **예외 처리 (Heuristics):**
    - 시각적으로 페이지 상단 15% 이내에 위치한 경우
    - DOM 구조상 부모의 첫 5번째 이내 자식으로 최상위 그룹에 속하는 경우
  - 결과: **[수정 권고]**
  - 가이드: "가급적 DOM 구조를 통해 순서를 제어할 것을 권장합니다."

### [검사 3] 수동 검사 안내 — 초점 이동 순서
- **Rule 2.1.2 (Focus Order):** 포커스 가능 요소 총 개수를 포함한 수동 검사 안내
  - 결과: **[검토 필요]** (항상 1건 발행)
  - 가이드: "상단 툴바의 '초점 순서 시각화' 버튼을 켠 후 Tab 키로 페이지를 탐색하세요. 초점이 시각적 배치 및 콘텐츠의 논리적 흐름과 일치하는 순서로 이동하는지 확인하고, 결과를 적절/오류로 판정한 뒤 점수를 직접 입력하세요."

### [검사 4] 수동 검사 안내 — 초점 아웃라인 표시
- **Rule 2.1.2 (Focus Visibility):** 포커스 가능 요소 총 개수를 포함한 수동 검사 안내
  - 결과: **[검토 필요]** (항상 1건 발행)
  - 가이드: "Tab 키로 각 요소를 순서대로 이동하면서 포커스 인디케이터(아웃라인, 배경색 변화 등)가 모든 요소에 명확하게 표시되는지 육안으로 확인하세요. 결과를 적절/오류로 판정한 뒤 점수를 직접 입력하세요."

> **자동화 한계:** `window.getComputedStyle(el)`은 요소가 현재 포커스를 받고 있지 않으면 `:focus` 상태의 스타일을 반환하지 않습니다. 따라서 인라인 스타일로 `outline: none`을 하드코딩한 극단적인 안티 패턴만 자동 탐지하고, 나머지는 수동 검사 안내(2건)로 유도합니다.

## 3. 최종 상태 정의 (Final Status)

1. **오류:** 인라인 스타일로 초점 표시가 강제로 제거된 경우.
2. **수정 권고:** 논리적 초점 이동 순서를 방해하는 양수 `tabindex`가 사용된 경우.
3. **검토 필요:** 전문가가 직접 Tab 키를 눌러 확인해야 하는 항목 (초점 순서, 아웃라인 표시 각 1건).

## 4. 점수 산출

자동 계산 불가 항목으로, 전문가가 각 수동 검사 항목을 적절/오류로 판정한 후 점수를 직접 입력한다.
- 두 항목 모두 이상 없음: 100점 권장
- 아웃라인 전체 미표시 등 중대 위반: 0점
- 일부 요소만 문제: 위반 비율에 따라 전문가 판단으로 입력
