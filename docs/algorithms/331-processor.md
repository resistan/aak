# ABT-3.3.1-Processor: 오류 정정 v0.11

### 🔗 References
- KWCAG 2.2: 3.3.1 오류 정정
- WCAG 2.2: 3.3.1 Error Identification (A) (Reference)

---

이 알고리즘은 **KWCAG 3.3.1(오류 정정)** 지침을 준수하며, 사용자가 입력 서식(Form)을 작성하는 과정에서 오류가 발생했을 때, 그 원인을 명확히 안내하고 정정할 수 있는 수단을 제공하는지 진단한다.

> **배점 방식:** 전수 조사형 (Exhaustive Scan). 필수 입력이 포함된 폼을 수집하여 각각 판정하며, 비율 기반으로 점수를 산정한다. 폼이 없는 경우에도 전문가 확인 요청 리포트를 1건 생성한다.

> **자동화 도구의 한계:** 브라우저 확장은 실제 폼 제출을 실행하거나 오류 상태를 강제로 유발할 수 없다. 정적 분석으로 폼 구조를 파악하고 전문가에게 수동 검사를 유도하는 방식을 채택한다.

## 1. 데이터 수집 단계 (Data Collection)

- **Forms:** `<form>` 요소 내에 `input[required]`, `select[required]`, `textarea[required]`, `[aria-required="true"]`가 1개 이상 포함된 경우
- **ARIA 오류 상태:** `aria-invalid="true"` 적용 요소 + `aria-describedby`·`aria-errormessage` 연결 여부

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [검사 1] 폼(Form) 스캔 및 수동 검사 유도

- **Rule 3.3.1 (Form Error Identification - Manual Check):** 필수 입력 항목이 포함된 폼 발견 시.
  - 결과: **검토 필요**
  - 가이드: "필수 입력 항목이 포함된 서식(form)입니다. 고의로 값을 비우거나 틀리게 입력한 후 폼을 제출해보세요. 오류 원인이 텍스트로 명확히 안내되고, 초점이 오류 항목으로 이동하는지 수동으로 확인해야 합니다."

- **Rule 3.3.1 (Manual Review):** 서식이 전혀 없는 경우에도 커스텀 위젯 확인을 위해 검토 필요 리포트 1건 생성.
  - 결과: **검토 필요**
  - 가이드: "서식(form) 요소는 없으나 입력 위젯이 있다면, 입력 오류 발생 시 그 원인과 정정 방법을 사용자에게 명확히 알려주는지 수동으로 검토하세요."

### [검사 2] ARIA 오류 연결 검사

- **Rule 3.3.1 (Missing Error Message Connection):** `aria-invalid="true"`로 오류 상태가 렌더링된 항목이 있으나, 구체적인 오류 메시지(`aria-errormessage` 또는 `aria-describedby`)가 연결되지 않은 경우.
  - 결과: **오류**
  - 가이드: "aria-invalid='true'로 오류 상태가 렌더링된 항목이 있으나, 구체적인 오류 메시지(aria-errormessage 또는 aria-describedby)가 연결되지 않았습니다."

- **Rule 3.3.1 (Verify ARIA Error Connection):** 오류 상태와 오류 메시지가 ARIA 속성으로 연결된 경우.
  - 결과: **검토 필요**
  - 가이드: "오류 상태(aria-invalid)와 오류 메시지가 ARIA 속성으로 연결되어 있습니다. 실제로 폼을 제출했을 때 화면에 오류 텍스트가 잘 보이고 초점이 이동하는지 최종 확인하세요."

### [단계 B] 전문가 수동 검사 체크리스트

전문가가 직접 폼을 잘못 입력·제출하여 확인해야 하는 항목:

1. **오류 원인 텍스트 안내** — 오류 원인과 정정 방법이 텍스트로 표시되는가? (빨간 테두리만이면 부적절)
2. **초점 이동** — 오류 확인 후 키보드 초점이 오류 항목으로 이동하는가?
3. **입력값 보존** — 오류 발생 시 이전에 입력한 다른 값들이 사라지지 않는가?

### [단계 C] 외부 시스템 검증 예외

- 은행 뱅킹의 계좌 확인, 인증서 검증, 결제망 승인 등 외부 시스템으로부터 처리 가능 여부를 검증받는 경우는 본 지침의 오류 정정 대상에서 제외한다.
- 단, 클라이언트 단에서 1차적으로 검증 가능한 값의 누락·형식(이메일 패턴 등)·범위 유효성 오류는 반드시 정정 방법을 제공해야 한다.
- 판정 기준: 외부 서버 응답에 의존하는 금융 결제 오류 등은 **[해당 없음(예외)]** 처리.

## 3. 최종 상태 정의 (Final Status)

1. **오류:** `aria-invalid="true"` 상태에서 오류 메시지 텍스트 미연결.
2. **검토 필요:** 필수 입력 포함 폼 또는 폼 미감지 시. 전문가 수동 제출 테스트 필요.
3. **적절:** 전문가가 오류 원인 안내·초점 이동·입력값 보존이 모두 정상임을 확인한 경우.
