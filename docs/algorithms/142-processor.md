# ABT-1.4.2-Processor: 자동 재생 금지 v0.13

### 🔗 References
- KWCAG 2.2: 1.4.2 자동 재생 금지
- WCAG 2.2: 1.4.2 Audio Control (A) (Reference)

---

이 알고리즘은 지침 1.4.2(자동 재생 금지)를 준수하며, 페이지 로드 시 자동으로 재생되는 음성 정보가 사용자에게 방해를 주지 않고 제어 가능한지 진단한다.

> **자동화 도구의 한계:** 브라우저의 보안 정책(CORS) 및 동적 재생의 다양성으로 인해, 미디어 요소 발견 시 전문가의 수동 검토를 유도하는 방식을 취한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Media Elements:** `<audio>`, `<video>`, YouTube/Vimeo `<iframe>`.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### 미디어 요소 탐지
- **Rule 1.4.2 (Manual Audio Review):** 페이지 내에 소리를 재생할 수 있는 요소가 존재하는지 검사.

**대상:**
- `<video>` 요소
- `<audio>` 요소
- `iframe[src*="youtube.com"]`
- `iframe[src*="vimeo.com"]`

**결과:** **[검토 필요]**
**가이드:** "미디어 요소가 감지되었습니다. 페이지 로드 시 소리가 자동으로 재생되는지, 그리고 이를 제어할 수 있는 수단이 있는지 수동으로 확인하세요."

## 3. 최종 상태 정의 (Final Status)

1. **검토 필요:** 페이지 내에 미디어 요소가 존재하여 자동 재생 여부 및 제어 수단 유무의 육안 확인이 필요한 상태.
2. **적절:** 전문가가 '검토 필요' 항목을 확인 후 자동 재생이 없거나 제어 수단이 충분함을 판정한 경우.
3. **N/A:** 페이지 내에 오디오/비디오 관련 요소가 전혀 없는 경우.
