# ABT-1.4.3-Processor: 텍스트 콘텐츠의 명도 대비 v0.13

### 🔗 References
- KWCAG 2.2: 1.4.3 텍스트 콘텐츠의 명도 대비
- WCAG 2.2: 1.4.3 Contrast (Minimum) (AA) (Reference)

---

이 알고리즘은 **KWCAG 1.4.3(텍스트 콘텐츠의 명도 대비)** 지침을 준수하며, 웹 페이지의 모든 텍스트 콘텐츠가 배경색과 충분한 대비를 이루어 저시력 사용자나 고령자도 정보를 명확히 인지할 수 있는지 진단한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Text Attributes:** 글자색(`color`), 폰트 크기(`font-size`), 굵기(`font-weight`).
- **Background Color:** 부모 요소를 거슬러 올라가며 실제 렌더링 배경색 추출 (투명 배경 처리).
- **Exceptions:** 이미지 대체 기법(IR)이 적용된 요소, 시각적으로 숨겨진 요소.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 텍스트 크기별 기준 설정

**일반 텍스트:**
- 조건: 24px 미만 또는 굵은(bold/700+) 18.66px 미만
- 최소 대비 기준: **4.5 : 1**

**큰 텍스트:**
- 조건: 24px 이상 또는 굵은(bold/700+) 18.66px 이상
- 최소 대비 기준: **3 : 1**

### [단계 B] 명암비 계산
- WCAG 2.x 상대 휘도(Relative Luminance) 공식 사용
- 공식: `(L1 + 0.05) / (L2 + 0.05)` (L1 > L2)

### [단계 C] 결과 판정
- **Rule 1.4.3 (Contrast Ratio):** 기준 미달 시
  - 결과: **[수정 권고]**
  - 메시지: "텍스트와 배경의 명도 대비가 X.XX:1로, 기준치(Y:1)보다 낮습니다. (폰트: Npx, weight)"

### [단계 D] 전수 조사 요약
- **Rule 1.4.3 (Contrast Valid):** 통과한 텍스트 요소는 개별 리포트 대신 **요약 리포트**로 집계
- `passCount` 필드에 통과 항목 수 기록
- UI 점수 계산 시 가중치로 활용

### [예외 처리]
- **IR 기법:** 이미지 대체 기법으로 시각적으로 숨겨진 텍스트는 검사 제외
- **숨겨진 요소:** `isHidden()` 판정 요소는 검사 제외

## 3. 최종 상태 정의 (Final Status)

1. **수정 권고:** 텍스트 명암비가 기준(4.5:1 또는 3:1)에 미달함.
2. **적절:** 모든 텍스트 콘텐츠가 표준 명암비 기준을 충족함.

## 4. 점수 계산 특이사항 (Scoring)
이 지침은 **전수 조사형(Exhaustive Scan)** 모델을 사용합니다:
- 페이지 내 모든 텍스트 요소를 검사
- 통과/미달 비율로 점수 산출
- `passCount`를 포함한 요약 리포트가 점수 계산의 분모 확장에 기여
