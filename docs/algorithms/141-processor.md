# ABT-1.4.1-Processor: 색에 무관한 콘텐츠 인식 v0.12

### 🔗 References
- KWCAG 2.2: 1.4.1 색에 무관한 콘텐츠 인식
- WCAG 2.2: 1.4.1 Use of Color (A) (Reference)

---

이 알고리즘은 지침 1.4.1(색에 무관한 콘텐츠 인식)을 준수하며, 정보의 전달, 동작의 지시, 응답의 유도 또는 시각적 요소의 구분이 오직 색상만으로 이루어지지 않는지 진단한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Links:** `<a>` 요소의 `color`, `text-decoration`, 부모와의 색상 차이.
- **Layout Context:** 부모의 `display` (flex/grid 여부), 직계 텍스트 노드 존재 여부.
- **Background Elements:** `div`, `span`, `i`, `b` 요소의 `background-color`, 텍스트/ARIA 레이블 유무.
- **Graphic Elements:** `<canvas>`, `<svg>`, `.chart`, `.graph`, `[id*="chart"]`, `[id*="graph"]` 등.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [검사 1] 링크 구분 검사
- **Rule 1.1 (Color Independence):** 본문 내 링크가 주변 텍스트와 오직 색상만으로 구분되는지 검사.

**조건:**
- 밑줄(`text-decoration: underline`)이 없음
- 부모와 색상이 다름
- 직계 텍스트 노드와 섞여 있음 (문장의 일부)
- 부모가 Flex/Grid가 아님 (독립 UI 항목 제외)
- 링크가 인라인 상태 (block/inline-block 제외)

**결과:** **[검토 필요]**
**가이드:** "문장이나 텍스트 뭉치 안에 포함된 링크가 밑줄 없이 색상으로만 구분되고 있습니다. 색을 인지하지 못하는 사용자를 위해 밑줄 등 추가적인 시각적 구분을 제공하세요."

### [검사 2] 배경색 의존성 검사
- **Rule 1.1 (Color Independence):** 텍스트나 레이블 없이 배경색만으로 상태나 정보를 전달하는 요소 탐지.

**조건:**
- 배경색이 투명이 아님
- 내부 텍스트가 없음
- `aria-label` 또는 `title`이 없음
- 크기가 0보다 큼

**결과:** **[검토 필요]**
**가이드:** "요소에 배경색은 있으나 텍스트나 레이블이 없습니다. 색상만으로 정보를 전달하고 있다면 패턴이나 텍스트를 추가하세요."

### [검사 3] 시각적 데이터 콘텐츠 검사
- **Rule 1.1 (Color Independence):** 그래프, 차트 등 색상으로 데이터를 구분하는 요소 탐지.

**대상:** `<canvas>`, `<svg>` (role="img" 제외), `.chart`, `.graph`, `[id*="chart"]`, `[id*="graph"]`

**조건:**
- 요소 크기가 50x50px 이상 (아이콘 제외)
- 중첩된 요소 중복 방지

**결과:** **[검토 필요]**
**가이드:** "그래프나 차트 등 시각적 정보를 담은 콘텐츠가 탐지되었습니다. 데이터의 계열이나 값을 구분할 때 색상뿐만 아니라 패턴, 모양, 레이블 등 색에 무관하게 인식할 수 있는 수단이 함께 제공되는지 검토하세요."

## 3. 최종 상태 정의 (Final Status)

1. **검토 필요:** 그래프/차트 콘텐츠, 밑줄 없는 링크, 배경색만 있는 요소 등 전문가의 정성적 판단이 필요한 경우.
2. **적절:** 색상 외에도 형태, 패턴, 텍스트 등을 통해 정보를 명확히 중복 전달함.
3. **N/A:** 색상 구분이 필요한 콘텐츠가 존재하지 않는 경우.
