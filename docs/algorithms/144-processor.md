# ABT-1.4.4-Processor: 콘텐츠 간의 구분 v0.12

### 🔗 References
- KWCAG 2.2: 1.4.4 콘텐츠 간의 구분
- WCAG 2.2: N/A (KWCAG 전용 지침)

---

이 알고리즘은 **KWCAG 1.4.4(콘텐츠 간의 구분)** 지침을 준수하며, 웹 페이지의 이웃한 콘텐츠가 테두리, 구분선, 배경색 명도 차, 간격 등을 통해 시각적으로 명확하게 구분되는지 진단한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Visual Styles:** 요소의 `border`, `border-style`, `background-color`.
- **Form Controls:** `input`, `select`, `textarea` 요소의 시각적 경계 정보.
- **Buttons:** `button` 요소의 위치 및 간격.
- **Parent Context:** 부모 요소의 배경색.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [검사 1] 입력 서식 구분 검사
- **Rule 1.4.4 (Visual Distinction):** 입력 필드가 배경과 시각적으로 구분되는지 검사.

**대상:** `input`, `select`, `textarea`

**조건:**
- 테두리가 없음 (`border-width: 0px` 또는 `border-style: none`)
- 배경색이 부모와 동일하거나 투명

**결과:** **[수정 권고]**
**가이드:** "입력 필드가 배경과 시각적으로 잘 구분되지 않습니다. 테두리나 배경색을 달리하여 입력 영역을 명확히 구분하세요."

### [검사 2] 버튼 간격 검사
- **Rule 1.4.4 (Visual Distinction):** 인접한 버튼 사이의 간격을 측정.

**조건:**
- 동일 부모 내의 연속된 버튼
- 수평 간격이 2px 미만
- 양쪽 버튼 모두 너비가 0보다 큼

**결과:** **[검토 필요]**
**가이드:** "인접한 버튼 간의 간격이 매우 좁습니다. 시각적 구분선이나 충분한 여백이 있는지 확인하세요."

### [검사 3] 레이아웃 블록 구분 검사
- **Rule 1.4.4 (Layout Differentiation):** 주요 영역의 시각적 구분 상태를 전역적으로 검토 유도.

**대상:** 문서 전체 (`body`)

**결과:** **[검토 필요]** (항상 1개 리포트 생성)
**가이드:** "웹 페이지의 주요 영역(헤더, 본문, 사이드바 등)이 테두리, 구분선, 배경색 또는 여백을 통해 시각적으로 명확하게 구분되는지 확인하세요."

## 3. 최종 상태 정의 (Final Status)

1. **수정 권고:** 입력 필드 등 중요 컨트롤의 시각적 경계가 약한 경우.
2. **검토 필요:** 레이아웃 영역이나 인접 버튼 간의 시각적 구분이 전문가의 정성적 확인이 필요한 수준인 경우.
3. **적절:** 모든 이웃한 콘텐츠가 다양한 시각적 단서를 통해 명확히 구분됨.
4. **N/A:** 페이지 내에 구분할 대상 콘텐츠가 존재하지 않는 경우.
