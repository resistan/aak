# ABT-2.4.1-Processor: 반복 영역 건너뛰기 v0.2

### 🔗 References
- KWCAG 2.2: 2.4.1 반복 영역 건너뛰기
- WCAG 2.2: 2.4.1 Bypass Blocks (A) (Reference)

---

이 알고리즘은 **KWCAG 2.4.1(반복 영역 건너뛰기)** 지침을 준수하며, 페이지 상단에 본문으로 바로 이동할 수 있는 '건너뛰기 링크(Skip Navigation)'의 유효성을 진단한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Focusable Elements:** 숨김 요소를 제외한 모든 포커스 가능 요소의 순서.
- **Skip Link Group:** 포커스 순서에서 첫 번째 `a[href^="#"]`가 5번째 이내에 위치하면 그 위치부터 연속된 `a[href^="#"]` 전체를 그룹으로 수집.
- **Target Elements:** 각 링크가 가리키는 대상 ID의 존재 여부 및 가시성.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [검사 1] 건너뛰기 링크 그룹 존재 및 위치 검사
- **Rule 2.4.1 (Missing Skip Link):** 포커스 순서 5번째 이내에 `a[href^="#"]`가 없는 경우.
  - 결과: **[오류]**
  - 가이드: "문서 최상단에 본문으로 바로가기(건너뛰기) 링크가 제공되지 않았습니다."

### [검사 2] 그룹 내 각 링크 유효성 검사 (링크별 개별 리포트)
- **Rule 2.4.1 (Empty Link Text):** 링크의 텍스트와 `aria-label`이 모두 비어있는 경우.
  - 결과: **[오류]**
- **Rule 2.4.1 (Invalid Target):** 링크의 대상 ID가 문서에 존재하지 않는 경우.
  - 결과: **[오류]**
- **Rule 2.4.1 (Hidden Target):** 링크의 대상 요소가 숨겨져 초점을 받을 수 없는 경우.
  - 결과: **[수정 권고]**
- 위 오류 없음:
  - 결과: **[적절]**

### [검사 3] 포커스 시 가시성 (수동 검사 안내)
- **Rule 2.4.1 (Focus Visibility):** 건너뛰기 링크 자체가 Tab 초점을 받을 때 화면에 표시되는지 자동 판별 불가.
  - 결과: **[검토 필요]** (항상 1건 발행, 그룹 전체에 대한 안내)
  - 가이드: "Tab 키로 페이지에 진입했을 때 건너뛰기 링크가 화면에 표시되는지 직접 확인하세요."

> **그룹 수집 로직:** `allFocusable` 순서에서 첫 번째 `a[href^="#"]` 발견 시 그룹 시작, 이후 연속된 `a[href^="#"]`를 모두 수집하며 다른 요소가 끼면 그룹 종료. 첫 번째 링크가 인덱스 5 이상이면 그룹 없음으로 판단.

## 3. 최종 상태 정의 (Final Status)
1. **오류:** 건너뛰기 링크 부재, 대상 ID 부재, 또는 링크 텍스트 누락.
2. **수정 권고:** 링크 대상이 숨겨져 있는 경우.
3. **적절:** 유효한 건너뛰기 링크가 최상단에 적절히 제공됨.
4. **검토 필요:** 포커스 시 링크 가시성 전문가 직접 확인 필요.

## 4. 점수 산출 (전수 조사형)

그룹 내 각 링크 + 포커스 가시성 검사 항목을 전수 조사형으로 계산.

- 포커스 가시성 항목을 전문가가 **적절** 판정 시 → 전체 100점 가능
- 포커스 가시성 항목을 **오류** 판정 시 → 해당 가중치만큼 감점
- 건너뛰기 링크 자체가 없으면 → 0점
