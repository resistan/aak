# ABT-2.4.1-Processor: 반복 영역 건너뛰기 v0.12

### 🔗 References
- KWCAG 2.2: 2.4.1 반복 영역 건너뛰기
- WCAG 2.2: 2.4.1 Bypass Blocks (A) (Reference)

---

이 알고리즘은 **KWCAG 2.4.1(반복 영역 건너뛰기)** 지침을 준수하며, 페이지 상단에 본문으로 바로 이동할 수 있는 '건너뛰기 링크(Skip Navigation)'의 유효성을 진단한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Skip Links:** `<a>` 태그의 `href`가 `#`으로 시작하는 내부 링크.
- **Focusable Elements:** 문서 내 모든 포커스 가능한 요소의 순서.
- **Target Elements:** 링크가 가리키는 대상 ID의 존재 여부 및 가시성.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 건너뛰기 링크 존재 여부 및 위치 검사
- **Rule 2.4.1 (Missing Skip Link):** 
  - 문서 초기 포커스 가능한 요소 5개 중 건너뛰기 링크가 없는 경우.
  - 결과: **[오류]**.

### [단계 B] 링크 유효성 및 대상 검사
- **Rule 2.4.1 (Invalid Target):** 
  - 링크의 대상 ID가 문서에 존재하지 않는 경우.
  - 결과: **[오류]**.
- **Rule 2.4.1 (Hidden Target):** 
  - 링크의 대상 요소가 숨겨져 있어 초점을 받을 수 없는 경우.
  - 결과: **[수정 권고]**.
- **Rule 2.4.1 (Empty Link Text):** 
  - 링크의 텍스트나 `aria-label`이 모두 비어있는 경우.
  - 결과: **[오류]**.

## 3. 최종 상태 정의 (Final Status)
1. **오류:** 건너뛰기 링크 부재, 대상 ID 부재, 또는 링크 텍스트 누락.
2. **수정 권고:** 링크 대상이 숨겨져 있는 경우.
3. **적절:** 유효한 건너뛰기 링크가 최상단에 적절히 제공됨.
