# ABT-3.3.4-Processor: 반복 입력 정보 v0.10

### 🔗 References
- KWCAG 2.2: 3.3.4 반복 입력 정보
- WCAG 2.2: 3.3.8 Redundant Entry (A) (Reference)

---

이 알고리즘은 **KWCAG 3.3.4(반복 입력 정보)** 지침을 준수하며, 사용자가 동일한 프로세스(예: 회원가입-주소 입력, 주문서 작성-배송지 입력) 내에서 이전에 입력했던 정보를 다시 입력해야 하는 경우, 자동 입력(Autofill) 또는 선택 입력(Select from previous)과 같은 편리한 방법을 제공하는지 진단한다.
> 💡 **단일 페이지 검사 원칙 및 ABT 엔진 역할**
> ABT 엔진(브라우저 확장 프로그램)은 현재 로드된 단일 페이지의 DOM만을 분석하는 것을 원칙으로 한다. 따라서 여러 페이지에 걸친 "동일 프로세스" 내 반복 입력 필드의 실제 재입력 여부 및 대체 수단 제공 여부를 엔진이 자동으로 판단하는 것은 불가능하며, **탐지된 반복 가능 필드에 대해 전문가의 수동 검사를 강하게 유도**하는 방향으로 동작한다.

## 1. 대상 식별 및 데이터 수집 (ABT 엔진 역할)

### [대상 요소 추출]
- **데이터 입력 필드**: `<input>`, `<textarea>`, `<select>` 등 사용자로부터 정보를 받는 모든 필드.
- **관련 컨텍스트**: 필드 이름, `id`, `name`, `aria-label` 등을 기반으로 현재 페이지에서 이전에 입력되었을 가능성이 있는 필드.
- **속성**: `autocomplete` 속성 존재 및 값.
- **UI 요소**: "이전 주소와 동일", "회원 정보와 동일" 등의 체크박스 또는 버튼.

## 2. 분석 및 분류 로직 (단일 페이지 분석 및 전문가 가이드)

### [검사 1] 반복 가능 입력 필드 탐지 및 대체 수단 검사
- **Rule 3.3.4 (Missing Autocomplete & Checkbox):** 반복 입력 의심 필드가 발견되었으나 `autocomplete` 속성이 없고 '이전 정보와 동일' 체크박스도 없는 경우.
  - 키워드: `name`, `이름`, `email`, `이메일`, `phone`, `tel`, `전화`, `address`, `주소`, `zip`, `우편번호`
  - 결과: **[검토 필요]**
  - 가이드: "[수동 검사 안내] 이름, 연락처, 주소 등을 입력하는 필드가 발견되었으나 autocomplete 속성이 없으며 '이전 정보와 동일' 체크박스도 보이지 않습니다. 만약 다단계 폼(주문 등)에서 이전 단계에 입력했던 정보를 다시 묻는 것이라면 오류입니다. 수동으로 프로세스를 확인하세요."

### [검사 2] 대체 수단 동작 확인
- **Rule 3.3.4 (Verify Auto-population):** 반복 입력 필드에 `autocomplete` 속성이나 '이전과 동일' 옵션이 발견된 경우.
  - 결과: **[검토 필요]**
  - 가이드: "[수동 검사 안내] 반복 입력 필드에 autocomplete 속성이나 '이전과 동일' 옵션이 발견되었습니다. 실제로 폼을 작성할 때 이전 정보가 잘 채워지는지 수동으로 확인하세요."

### [검사 3] 반복 입력 폼 부재 처리
- **Rule 3.3.4 (No Suspect Forms):** 페이지 내에서 반복 입력 의심 필드를 발견하지 못한 경우.
  - 결과: **[N/A]**
  - 가이드: "페이지 내에서 이름, 주소, 연락처 등 프로세스 간 반복 입력이 주로 발생하는 입력 서식을 발견하지 못했습니다. 단일 프로세스일 가능성이 높습니다."

### [단계 D] 전문가 수동 검사 가이드라인
도구는 전문가에게 다음 체크리스트를 제공하여, "동일 프로세스" 내에서 반복 입력이 발생하는 시나리오를 직접 수행하고, 자동/선택 입력 기능이 올바르게 작동하는지 수동으로 검증하도록 유도한다.
- *체크리스트*:
  - **반복 입력 시나리오 수행**: 이전 페이지/단계에서 특정 정보를 입력한 후, 현재 페이지에서 동일한 정보의 재입력을 요구하는가?
  - **자동 완성 기능 확인**: 브라우저의 자동 완성 기능 또는 `autocomplete` 속성에 의해 정보가 자동으로 채워지는가?
  - **선택 입력 UI 확인**: "이전 정보와 동일" 등의 UI가 클릭 시 정보를 채워주는가?
  - **보안 예외 확인**: 비밀번호 등 보안상 반복 입력이 필요한 필드였는가?
