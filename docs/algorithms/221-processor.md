# ABT-2.2.1-Processor: 응답 시간 조절 v0.11

### 🔗 References
- KWCAG 2.2: 2.2.1 응답 시간 조절
- WCAG 2.2: 2.2.1 Timing Adjustable (A) (Reference)

---

이 알고리즘은 **KWCAG 2.2.1(응답 시간 조절)** 지침을 준수하며, 자동 새로고침이나 세션 만료 등 시간 제한이 있는 기능을 사용자가 제어할 수 있는지 진단한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Meta Refresh:** `<meta http-equiv="refresh">` 태그의 `content` 속성 분석.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [검사 1] 자동 새로고침 탐지
- **Rule 2.2.1 (Meta Refresh):** `<meta http-equiv="refresh">` 태그가 존재하고 `content` 값이 0보다 큰 숫자로 시작하는 경우.
  - 결과: **[오류]**
  - 가이드: "<meta http-equiv=\"refresh\"> 태그를 사용한 자동 새로고침/리다이렉트가 감지되었습니다. 사용자가 이를 제어할 수 없습니다."

### [검사 2] 수동 검토 안내
- **Rule 2.2.1 (Manual Review):** 메타 태그 이외의 JavaScript 기반 시간 제한(세션 만료 알림, 팝업 자동 닫힘 등)은 정적 분석으로 탐지가 어려우므로 수동 검토를 유도함.
  - 결과: **[검토 필요]**
  - 가이드: "페이지 내에 자동 새로고침(meta refresh) 이외의 시간 제한(세션 만료, 팝업 자동 닫힘 등)이 있다면, 사용자가 시간을 연장하거나 정지할 수 있는 수단이 제공되는지 수동으로 확인하세요."

## 3. 최종 상태 정의 (Final Status)
1. **오류:** `<meta>` 태그를 이용한 강제 새로고침/리다이렉트가 설정된 경우.
2. **검토 필요:** 페이지 내에 존재할 수 있는 동적 시간 제한 요소에 대한 전문가 확인이 필요한 상태.
