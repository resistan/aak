# ABT-2.4.2-Processor: 제목 제공 v0.12

### 🔗 References
- KWCAG 2.2: 2.4.2 제목 제공
- WCAG 2.2: 2.4.2 Page Titled (A) (Reference)

---

이 알고리즘은 **KWCAG 2.4.2(제목 제공)** 지침을 준수하며, 페이지와 프레임에 해당 내용을 요약한 제목이 제공되는지, 그리고 문서의 헤딩 구조가 논리적인지 진단한다.

## 1. 데이터 수집 단계 (Data Collection)
- **Page Title:** `<head>` 내의 `<title>` 요소.
- **Frame Titles:** `<iframe>`, `<frame>`의 `title` 속성.
- **Headings:** `<h1>`~`<h6>` 요소의 계층 구조.

## 2. 분석 및 분류 로직 (Analysis Pipeline)

### [단계 A] 페이지 및 프레임 제목 검사
- **누락 체크:** `<title>`이 없거나 비어있는 경우, 또는 프레임의 `title` 속성이 없는 경우.
  - 결과: **[오류]**.
- **의미 없는 제목:** "Untitled", "새 탭", "제목 없음", "iframe" 등 무의미한 기본값 사용 시.
  - 결과: **[부적절]**.
- **내용 검토:** 제목이 존재하더라도 적절성 확인을 위해 전문가 검토 유도.
  - 결과: **[검토 필요]**.

### [단계 B] 헤딩 구조 분석 (Heading Outline)
- **H1 존재 여부:** 페이지 내에 `<h1>`이 없는 경우.
  - 결과: **[검토 필요]**
  - 가이드: "페이지 내에 대주제(<h1>)가 존재하지 않습니다. 문서의 핵심 주제가 적절하게 식별되는지 검토하세요."
- **헤딩 아웃라인 수집:** 문서의 `<h1>`~`<h6>` 구조를 수집하여 참고자료로 제공.
  - 결과: **[참고자료]**
- **헤딩 순서 논리성:** 헤딩 수준을 건너뛰는 경우 (예: h1 다음 h3).
  - 결과: **[수정 권고]**
  - 가이드: "헤딩 수준을 순차적으로 사용하는 것을 권장합니다 (예: h1 -> h2 -> h3)."

## 3. 최종 상태 정의 (Final Status)
1. **오류:** 페이지 또는 프레임 제목 누락.
2. **부적절:** 의미 없는 제목(기본값) 사용.
3. **검토 필요:** 제목의 적절성 확인 또는 `<h1>` 부재.
4. **수정 권고:** 헤딩 계층 구조의 논리적 순서 위배.
5. **참고자료:** 페이지 헤딩 구조(Heading Outline) 분석 결과.
6. **적절:** 제목이 제공되고 `<h1>`이 존재하는 상태.
