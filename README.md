# AAK (A11Y Assistant for KWCAG) Workbench

현재 개발 단계에 있으며 오류가 발생할 수 있습니다.

> 🛡️ **한국 웹 접근성 전문가를 위한 정밀 진단 어시스턴트**
> 브라우저 사이드 패널(Side Panel)을 통해 실시간 접근성 진단 및 전문가 판정 프로세스를 통합합니다.


## 💡 핵심 철학 (Core Philosophy)
웹 접근성 검수는 자동화하기 어려운 '**인적 판단**'의 영역이 많습니다. AAK는 "100% 자동화"를 목표로 하지 않습니다. 대신 "**전문가가 판단해야 할 항목을 가장 빠르게 선별해주고, 판단에 필요한 모든 정보(지침 원문, 마크업, 위치)를 한곳에 모아주는 것**"에 집중합니다.

## 🚀 주요 기능
- **KWCAG 2.2 최적화 진단**: 국내 표준 지침(33개 항목)에 특화된 진단 엔진 탑재
- **Expert-in-the-loop 판정**: 자동 진단 결과에 대해 전문가가 즉시 '적절/오류'를 판정하고 소견을 기록하는 인터페이스 제공
- **SVG Spotlight**: 위배 요소의 위치를 정확히 찾아주는 고해상도 시각적 피드백 제공
- **검사항목별 전용 도구**: 대체 텍스트 오버레이(1.1.1), CSS 선형화(1.3.2), 명도 대비 분석기(1.4.3), 초점 순서 시각화(2.1.2) 등
- **점수 배지**: 지침별 자동 점수 산출 및 수동 점수 입력 지원
- **세션 관리**: 과거 진단 기록 보관 및 열람, 개별/전체 삭제
- **창 분리**: 사이드 패널 ↔ 독립 창 전환으로 멀티 모니터 활용
- **스마트 리포팅**: 검수 내역을 마크다운(Markdown)으로 추출
- **Hybrid Audit Flow**: 자동 진단이 놓친 정성적 위배 사항을 전문가가 지침별로 직접 추가하여 리포트에 포함 가능

### 0.6.8 변경 내역
- 해당 URL이 활성화되지 않은 상태에서 기존 결과를 볼 수 없는 문제 수정
- 검사 결과 목록 중 N/A, 수동 입력, 정보 버튼 명도대비 조정
- 검출된 항목의 셀렉터 대신 실제 태그 내용 일부를 출력하도록 변경(셀렉터 출력은 fallback)
- 2.4.3 적절한 링크 텍스트의 오류 검출 방식 수정

## 🛠️ 기술 스택
- **Core**: React 18, TypeScript, SCSS (Module)
- **Extension**: Manifest V3, Chrome Side Panel API
- **Engine**: Custom JS Engine (`ABTCore`) + 33 Processors
- **Build**: Vite 4
- **State**: Zustand (Custom Persistence Middleware)
- **Icons**: Lucide React

## 📦 설치 및 실행
```bash
# 의존성 설치
npm install

# 프로덕션 빌드 (dist/ 폴더 생성)
npm run build
```

## 🔌 확장 프로그램 로드 방법
1. Chrome 브라우저에서 `chrome://extensions/` 주소로 이동합니다.
2. 우측 상단의 **'개발자 모드'**를 활성화합니다.
3. **'압축해제된 확장 프로그램을 로드합니다'** 버튼을 클릭합니다.
4. 프로젝트의 `dist` 폴더를 선택합니다.
5. 브라우저 툴바의 확장 프로그램 아이콘을 클릭하여 **AAK**를 실행하면 사이드 패널이 열립니다.

## 📂 프로젝트 구조
- `src/engine/`: 접근성 진단 코어 엔진 및 지침별 프로세서 (Content Script)
- `src/renderer/`: React 기반 Side Panel UI 대시보드
  - `components/`: UI 컴포넌트 (Header, Dashboard, WorkArea, Panels 등)
  - `hooks/`: 커스텀 훅 (useAudit, useSessionManager, useContrastPicker 등)
  - `utils/`: 유틸리티 함수 (format, standards, clipboard, color 등)
  - `store/`: Zustand 상태 관리
- `src/extension/`: 서비스 워커 및 익스텐션 배포 설정
- `docs/algorithms/`: 각 진단 알고리즘의 동작 원리 명세
- `.references/`: KWCAG 공식 지침 및 표준 데이터셋

## 🗺️ 로드맵 (Roadmap)
- **Phase 1 (완료)**: 진단 알고리즘 작성
- **Phase 2 (완료)**: 진단 프로세스 테스트
- **Phase 3 (완료)**: 일렉트론(폐기) -> 크롬 익스텐션 환경 전환
- **Phase 4 (완료)**: Batch 전송 아키텍처 도입
- **Phase 5 (완료)**: 33개 지침 알고리즘 구현 (KWCAG 2.2 표준 전 항목 진단 프로세서)
- **Phase 6 (완료)**: 표준 데이터셋 추가 (2025 인증평가 지침 JSON 데이터화 및 매핑)
- **Phase 7 (완료)**: 검사항목별 전용 도구 (CSS 선형화, Alt 오버레이 등 각 지침에 배치)
- **Phase 8 (완료)**: 정밀 진단 도구 고도화 (수동 의견 추가, 명도 대비 분석기, 초점 순서 시각화, 점수 배지, 세션 관리, 창 분리, 컴포넌트 분리)
- **Phase 9 (완료)**: 크롬 웹스토어 등록
- **Phase 10 (진행)**: 베타 테스트 (접근성 전문가 커뮤니티 피드백 수집 및 실무 환경 검증)
- **Phase 11 (계획)**: 성능 최적화 (Web Worker 기반 분석 엔진 전환 및 대용량 페이지 대응)
- **Phase 12 (계획)**: 회귀 방지 및 신뢰도 강화 (표준 시나리오 기반 자동 검증 체계 구축 및 사이드 패널 자체 접근성 완결)
- **Phase 13 (계획)**: 전문가 가이드라인 및 매뉴얼 정리
