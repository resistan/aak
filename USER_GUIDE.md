# AAK 사용자 가이드

A11Y Assistant for KWCAG(이하 AAK)는 한국 웹 접근성 지침(KWCAG 2.2)을 기반으로 웹 접근성 전문가의 검수 업무를 보조하기 위해 설계된 브라우저 확장 프로그램입니다. 본 가이드는 AAK의 설치부터 정밀 진단, 전문가 도구 활용 및 리포트 생성까지의 전 과정을 안내합니다.

---

## 빠른 시작

### 기본 검수 순서

1. **AAK 열기**: 툴바의 AAK 아이콘(돋보기 모양)을 클릭하여 사이드 패널을 엽니다.
2. **페이지 준비**: 검수할 페이지로 이동한 후 새로고침합니다.
3. **진단 실행**: 사이드 패널에서 **진단 시작** 버튼을 클릭합니다.
4. **결과 검토**: 지침별로 검출된 항목을 확인합니다.
   - 오류 항목: 즉시 수정이 필요한 위배 사항
   - 검토 필요 항목: 전문가 판단이 필요한 사항
5. **판정 수행**: 각 항목에 대해 **적절** 또는 **오류** 버튼으로 최종 판정합니다.
6. **리포트 추출**: 검수 완료 후 리포트 아이콘을 클릭하여 결과를 저장합니다.

### 보조 도구가 있는 검사항목

- **1.1.1 대체 텍스트**: 대체텍스트 보기 도구로 이미지 검수
- **1.3.2 의미 있는 순서**: CSS 선형화 도구로 읽기 순서 확인
- **1.4.3 명도 대비**: 대비 분석기로 텍스트 색상 검수
- **2.1.1~2.1.2 키보드 접근성**: 초점 순서 시각화로 탭 이동 검수

## 1. 설치 및 활성화

### 확장 프로그램 설치
[Chrome 웹 스토어](https://chromewebstore.google.com/detail/aak-workbench-kwcag/elkfaoflahgmpohidpefbijhnpncledj)에서 **AAK Workbench** 페이지를 열고 **Chrome에 추가** 버튼을 클릭합니다.

### 사이드 패널 열기
설치 후 브라우저 툴바의 **AAK 아이콘(돋보기 모양)**을 클릭하면 사이드 패널이 열립니다.

---

## 2. KWCAG 2.2 자동 진단

AAK는 KWCAG 2.2의 33개 지표를 자동으로 분석하여 전문가가 판단해야 할 핵심 요소를 선별합니다.

### 진단 실행
- 사이드 패널 상단의 **진단 시작(Run Audit)** 버튼을 클릭합니다.
- 엔진이 현재 페이지의 DOM을 분석하여 지침별 위배 사항을 추출합니다.

### 결과 해석
진단 결과는 다음과 같은 상태로 표시됩니다:
- **적절 (Pass)**: 자동 진단 기준을 통과한 항목입니다.
- **오류 (Fail)**: 지침 위배가 확실시되는 항목입니다.
- **검토 필요 (Manual Check)**: 알고리즘만으로는 판단이 불가능하여 전문가의 육안 확인이 필요한 항목입니다.

### 빠른 판정
각 진단 항목에는 **적절** / **오류** 버튼이 표시됩니다. 클릭 한 번으로 해당 항목의 최종 상태를 즉시 결정할 수 있어 대량의 항목을 빠르게 검토할 수 있습니다.

### 전문가 판정 (Expert-in-the-loop)
- 각 진단 항목을 클릭하면 상세 내역과 위배 요소의 마크업을 확인할 수 있습니다.
- **Spotlight** 기능으로 해당 요소의 위치가 화면에 강조 표시됩니다.
- **전문가 판정** 버튼을 클릭하여 상세 소견을 기록하고 최종 상태(적절/오류)를 결정할 수 있습니다.

### 수동 의견 추가
각 지침 헤더의 **+** 버튼을 클릭하면 자동 진단에서 검출되지 않은 항목에 대해 전문가가 직접 검토 의견을 추가할 수 있습니다.
- 항목 이름과 평가 소견을 입력합니다.
- 적절/오류 상태를 선택하여 저장합니다.

## 3. 전문가 전용 도구 활용

각 지침별로 검수 효율을 높여주는 전용 도구가 제공됩니다.

### 명도 대비 분석기 (1.4.3 지침)
- **명도 대비 케이스 추가** 버튼을 클릭합니다.
- **EyeDropper** 기능을 사용하여 화면상의 전경색과 배경색을 직접 추출합니다.
- 추출된 색상을 바탕으로 KWCAG 기준(4.5:1 또는 3:1) 충족 여부를 즉시 계산합니다.
- 측정 결과는 해당 지침의 진단 항목으로 저장됩니다.

### 대체 텍스트 오버레이 (1.1.1 지침)
- **대체텍스트 보기** 버튼을 클릭합니다.
- 페이지 내 모든 이미지 요소 위에 `alt` 속성값을 오버레이로 표시합니다.
- 이미지와 대체 텍스트의 적절성을 한눈에 비교 검수할 수 있습니다.

### CSS 선형화 (1.3.2 지침)
- **선형화(CSS 끄기)** 버튼을 클릭합니다.
- 페이지의 CSS를 일시적으로 제거하여 콘텐츠의 논리적 구조와 읽기 순서를 확인합니다.

### 포커스 트래커 (2.1.2 지침)
- **초점 순서 시각화** 버튼을 클릭합니다.
- 키보드 Tab 키 이동에 따른 초점 이동 순서를 SVG 경로로 시각화합니다.
- 논리적 초점 이동 순서가 위배되는 지점을 직관적으로 파악할 수 있습니다.
- **초기화** 버튼으로 경로를 리셋할 수 있습니다.

## 4. 점수 및 세션 관리

### 점수 배지
각 지침 헤더에 점수 배지가 표시됩니다:
- **자동 계산 점수**: 진단 결과를 바탕으로 자동 산출됩니다.
- **수동 검사 필요**: 자동 진단이 어려운 항목으로, 클릭하여 직접 점수를 입력할 수 있습니다.
- **N/A**: 해당 지침에 검출된 요소가 없는 경우입니다.

점수 배지를 클릭하면 0-100 사이의 점수를 직접 입력하여 수동으로 지정할 수 있습니다.

### 세션 관리
- 대시보드에서 **과거 진단 기록**을 확인할 수 있습니다.
- 각 세션을 클릭하면 해당 시점의 진단 결과를 다시 열람합니다.
- 휴지통 아이콘을 클릭하여 개별 세션을 삭제하거나, **전체 삭제** 버튼으로 모든 기록을 삭제할 수 있습니다.

## 5. 진단 리포트 익스포트

검수가 완료되면 결과물을 문서화하여 공유할 수 있습니다.

### 리포트 생성
- 헤더의 **리포트 추출** 아이콘을 클릭합니다.
- Markdown 파일로 저장하거나 클립보드에 복사됩니다.

### 리포트 활용
- 생성된 Markdown 파일에는 지침별 진단 결과, 전문가 소견, 위배 요소의 마크업 정보가 체계적으로 정리되어 있습니다.
- 이 파일은 GitHub Issue, Jira 티켓 또는 최종 검수 보고서의 기초 자료로 즉시 활용 가능합니다.

---

## 6. 창 분리 및 상태 유지

### 창 분리 (Pop-out)
- 헤더의 **창 분리** 아이콘을 클릭하면 사이드 패널이 독립 창으로 분리됩니다.
- 분리된 창에서 **사이드 패널로 돌리기** 버튼을 클릭하면 다시 사이드 패널로 전환됩니다.
- 멀티 모니터 환경에서 진단 결과를 별도 화면에 띄워놓고 작업할 때 유용합니다.

### 상태 유지
브라우저를 종료하거나 페이지를 새로고침해도 진행 중인 검수 내역과 전문가 소견은 안전하게 유지됩니다.
